# Contributing to Power Menu

Thank you for your interest in contributing! Here's how you can help.

## Code Style

- Follow the Flipper Zero coding conventions
- Use 4 spaces for indentation
- Add comments for complex logic
- Keep lines under 100 characters where practical
- Use descriptive variable and function names

## Making Changes

1. **Fork the repository** on GitHub

2. **Create a feature branch:**

```bash
git checkout -b feature/your-feature-name
```

3. **Make your changes** and commit with clear messages:

```bash
git commit -m "Add: description of your changes"
```

4. **Push to your fork:**

```bash
git push origin feature/your-feature-name
```

5. **Create a Pull Request** with a description of your changes

## Types of Contributions

### Bug Reports

- Describe the issue clearly
- Include steps to reproduce
- Specify your Flipper firmware version
- Include any error messages

### Feature Requests

- Explain the use case
- Describe how it would improve the app
- Provide mockups or examples if helpful

### Code Improvements

- Performance optimizations
- Code cleanup and refactoring
- Additional power options
- Better UI/UX

### Documentation

- Typo fixes
- Clearer explanations
- Additional examples
- Translation improvements

## Pull Request Process

1. Update documentation if needed
2. Test your changes thoroughly
3. Ensure code follows the style guide
4. Fill out the PR template completely
5. Keep PRs focused on a single feature or fix

## Testing

Before submitting:

1. Build the application: `./fbt fap_power_menu`
2. Test on actual Flipper hardware when possible
3. Test all menu options and confirm dialogs
4. Verify button inputs work correctly

## Questions?

- Open an issue for general questions
- Comment on relevant issues for specific discussions
- Check existing issues before creating duplicates
